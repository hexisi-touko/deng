要实现时间复杂度为O(n)的算法，核心是避免全排序（O(n log n)），而是通过**快速选择（Quickselect）** 找到数组中第k大的元素，再基于该元素划分出前k个最大元素（无需完全排序）。这种思路能将时间复杂度优化到O(n)（平均情况）。


### 算法设计思路
1. **确定子集大小**：设总元素数为n，两个子集的大小分别为`n1 = (n + 1) // 2`（较大子集）和`n2 = n // 2`（较小子集），确保`|n1 - n2|`最小（0或1）。
2. **快速选择找分界**：用快速选择算法找到数组中第n1大的元素（即排序后第n1个位置的元素），以此为界划分出前n1个最大元素（无需完全排序）。
3. **划分子集并求和**：基于分界元素，将数组分为“前n1个最大元素”（A1）和“剩余元素”（A2），分别计算它们的和，确保`|S1 - S2|`最大。



### 复杂度分析
- **时间复杂度**：快速选择算法的平均时间复杂度为O(n)（通过随机化pivot，最坏情况O(n²)的概率极低），后续划分和求和操作均为O(n)，因此整体平均时间复杂度为O(n)。
- **空间复杂度**：O(n)（主要用于存储输入数组的副本，递归调用栈的深度平均为O(log n)，可忽略）。


### 关键说明
快速选择算法通过“分区”操作，无需完全排序数组，只需确定第k大元素的位置，就能划分出前k个最大元素。这种思路避免了全排序的O(n log n)开销，实现了线性时间复杂度，适用于大规模数据场景。
